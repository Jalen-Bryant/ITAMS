# ITAMS Application Guide

ITAMS is an IT asset management application for tracking hardware inventory,
asset assignments, user access, reporting, and history. The production app is
hosted at `https://itams.app/`.

## What The App Manages

ITAMS is organized around these operational records:

| Area | Purpose |
| --- | --- |
| Assets | Tracks devices and equipment such as laptops, desktops, monitors, mobile devices, docks, printers, servers, scanners, and peripherals. |
| Assignments | Tracks which user has an asset, who assigned it, start/end dates, and notes. |
| Users | Tracks ITAMS user accounts, roles, department, active status, and login capability. |
| Reports | Provides operational summaries and charts for assets, users, assignments, warranty dates, and lifecycle activity. |
| History | Shows audit log entries and lifecycle events generated by app activity. |
| Account | Lets signed-in users change their own password and sign out. |

Asset status values are `InStock`, `Assigned`, `Active`, `InRepair`,
`Retired`, and `Lost`.

## Main Pages

| Page | Route | Description |
| --- | --- | --- |
| Login | `/login` | Signs users in with username/email and password. |
| Home | `/home` | Shows role-aware shortcuts into the app. |
| Assets | `/assets` | Lists, creates, edits, and deletes assets when the role permits it. |
| Assignments | `/assignments` | Lists, creates, edits, and deletes asset assignments when the role permits it. |
| Users | `/users` | Lists and manages users when the role permits it. |
| Reports | `/reports` | Shows KPI, breakdown, and time-series reporting views. |
| History | `/history` | Shows audit and lifecycle history. |
| Change Password | `/account/change-password` | Lets the current user update their password. |

The navigation menu hides links that the current user's role cannot read. The
API also enforces the same authorization policies server-side.

## Roles And Permissions

ITAMS currently recognizes these role names:

- `Admin`
- `Manager`
- `Technician`
- `User`
- `Auditor`

| Capability | Admin | Manager | Technician | Auditor | User |
| --- | --- | --- | --- | --- | --- |
| Sign in and change own password | Yes | Yes | Yes | Yes | Yes |
| View reports | Yes | Yes | No | Yes | No |
| View assets | Yes | Yes | Yes | Yes | No |
| Create, edit, delete assets | Yes | Yes | Yes | No | No |
| View assignments | Yes | Yes | Yes | Yes | No |
| Create, edit, delete assignments | Yes | Yes | Yes | No | No |
| View users | Yes | Yes | No | Yes | No |
| Create, edit, delete users | Yes | No | No | No | No |
| View history | Yes | Yes | No | Yes | No |

Use the least privileged role that supports the user's job. Reserve `Admin`
accounts for staff who need user management or full operational control.

## Authentication And Sessions

- Users sign in at `https://itams.app/login`.
- The API issues a JWT access token and refresh token after successful login.
- The client stores the session in browser local storage under
  `itams.auth.session`.
- Access tokens are short-lived. Refresh tokens let the client refresh the
  session while the backing server session remains active.
- Every authorized API request checks that the user's backing session is still
  active.
- Signing out revokes the current session.
- Changing a password revokes existing sessions for that user.
- Inactive users cannot continue using the application.

If a browser appears stuck with stale sign-in state, clear site data for
`itams.app` or remove the `itams.auth.session` local storage item and sign in
again.

## Common Workflows

### Add Or Update An Asset

1. Sign in with an `Admin`, `Manager`, or `Technician` account.
2. Open `Assets`.
3. Create a new asset or edit an existing one.
4. Enter the asset tag, serial number, type, manufacturer, model, status,
   department, location, lifecycle dates, and notes.
5. Save the change.

Asset changes write audit and lifecycle history where applicable.

### Assign An Asset

1. Sign in with an `Admin`, `Manager`, or `Technician` account.
2. Open `Assignments`.
3. Create or edit an assignment.
4. Select the asset, user, start date, optional end date, and notes.
5. Save the change.

Assignment changes update the asset's current assignment state and write audit
and lifecycle history.

### Manage Users

1. Sign in with an `Admin` account.
2. Open `Users`.
3. Create, edit, activate, deactivate, or delete users as needed.
4. Assign the lowest role that supports the user's responsibilities.

Only `Admin` can write user records. `Manager` and `Auditor` can read users.

### Review Reports

1. Sign in with an `Admin`, `Manager`, or `Auditor` account.
2. Open `Reports`.
3. Use the preset range or custom date filters.
4. Review KPIs, asset breakdowns, user breakdowns, assignment trends, warranty
   expiration trends, and lifecycle activity.

### Review History

1. Sign in with an `Admin`, `Manager`, or `Auditor` account.
2. Open `History`.
3. Review audit logs and lifecycle events.
4. Use IDs and timestamps to correlate changes with assets, users, and
   assignments.

## Support Checks

When a user reports a problem, first confirm:

- They are using `https://itams.app/`.
- Their account is active.
- Their role grants access to the page or action.
- Their browser does not have stale session data.
- The public smoke checks in [README.md](./README.md) pass.
